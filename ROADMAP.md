# Roadmap

## Vision

Make envcheck-cli the go-to tool for environment variable validation across all languages and platforms.

## Current Status: v1.0.0

✅ Core functionality complete
✅ Multi-language support (6 languages)
✅ CI/CD integration
✅ Zero dependencies

## Short Term (v1.1.0 - v1.3.0)

### v1.1.0 - Enhanced Configuration
- [ ] Support custom .env file names (.env.template, .env.sample)
- [ ] Configuration file support (.envcheckrc)
- [ ] Ignore patterns for specific variables
- [ ] Custom output templates

### v1.2.0 - Better Developer Experience
- [ ] Interactive mode for fixing issues
- [ ] Auto-generate .env.example from code
- [ ] Suggest fixes for common issues
- [ ] Better error messages with suggestions

### v1.3.0 - Performance & Scale
- [ ] Parallel file scanning
- [ ] Incremental scanning (only changed files)
- [ ] Cache results for faster subsequent runs
- [ ] Benchmark suite

## Medium Term (v2.0.0)

### Language Expansion
- [ ] PHP support
- [ ] Java/Kotlin support
- [ ] C#/.NET support
- [ ] Elixir support
- [ ] Swift support

### Advanced Features
- [ ] Detect unused env vars in .env files (not just .env.example)
- [ ] Validate env var naming conventions
- [ ] Check for sensitive data in code
- [ ] Integration with secret management tools (Vault, AWS Secrets Manager)

### Ecosystem Integration
- [ ] VS Code extension
- [ ] Pre-commit hook package

- [ ] GitHub App for automated PR comments

## Long Term (v3.0.0+)
